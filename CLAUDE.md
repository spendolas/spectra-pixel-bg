# CLAUDE.md

## Project Overview

**spectra-pixel-bg** is a generative art project that creates an interactive, animated pixel-based background visualization for spendolas.com. It uses WebGL-powered noise generation with Three.js, interactive presets, and color palettes to produce dynamic, procedurally-generated visual effects. Users cycle through presets and color schemes in real-time via keyboard controls.

Live site: https://www.spendolas.com/

## Tech Stack

- **JavaScript (ES6+)** with ES module syntax
- **Three.js r128** loaded via CDN (`cdnjs.cloudflare.com`)
- **spectraGL-noisekit** — custom WebGL visualization engine (unminified source in `src/`, ~1517 lines)
- **Terser** (or UglifyJS fallback) for minification during release
- **Python 3** for HTML text replacement in build scripts
- **No package manager, test framework, or linter**
- Deployed as a static site via **GitHub Pages** (CNAME: spendolas.com)

## Branch Strategy

- **`fork`** — Development branch. All edits happen in `src/`.
- **`main`** — Production/release branch. Contains root-level assets for GitHub Pages (generated from `dist/`).

## Running Locally

No build step needed for development. Serve `src/` with any HTTP server:

```bash
cd src && python -m http.server 8000
# or
npx http-server src/
```

Then visit `http://localhost:8000`.

## Release Process

```bash
# 1. Make changes in src/ on the fork branch
# 2. Build dist/ assets
./scripts/release.sh

# 3. Publish to main branch for GitHub Pages
./scripts/publish_release.sh
```

`release.sh` cleans `dist/`, copies source files, minifies `spectraGL-noisekit.js` via Terser, and rewrites HTML to reference the `.min.js`. `publish_release.sh` orchestrates the full release: runs `release.sh`, checks out `main`, syncs `dist/` contents to the repo root, commits with a timestamp, and pushes.

## Architecture

1. `index.html` loads Three.js (CDN), `spectraGL-noisekit.js`, then `main.js` (type="module")
2. `main.js` imports `{ PRESETS, PALETTES }` from `presets.js`
3. On DOM load, a random preset and palette are selected and deep-merged with `DEFAULT_OPTIONS`
4. `spectraGL({ target, ...mergedOptions })` initializes the WebGL visualization in `div#bg`
5. Keyboard events trigger preset/palette changes via `effect.updateOptions()`

### Key Globals

```js
window.__spectra        // Reference to the spectraGL effect instance
window.__chosenPreset   // Current preset object
window.__chosenPalette  // Current palette object
```

### spectraGL API

```js
const effect = spectraGL({ target: element, ...options });
effect.updateOptions(newOptions);  // Live update
effect.pause();
effect.play();
effect.destroy();
spectraGL.isSupported();           // WebGL feature detection
spectraGL.version;                 // "1.0.0"
```

## Key Patterns

- **Deep merge for config**: `DEFAULT_OPTIONS` -> preset options -> palette options
- **Random seed**: Regenerated on preset change, preserved on palette-only change
- **Safe list filtering**: Presets/palettes validated before use, graceful fallback if empty
- **Keyboard controls**: `r` = random all, `p` = random palette, `o` = random preset, `q` = toggle label
- **pickRandomExcluding()**: Prevents consecutive identical selections on rapid key presses

## Data Structures

### Presets (`src/presets.js`)

```js
// PRESETS: [{ name: string, options: object }]
```

Each preset's `options` can include: noiseType (`value`|`simplex`|`worley`|`box`), fractalType (`fbm`|`ridged`|`turbulence`|`none`), speed, meshDetail, foldIntensity, field settings, and more.

### Palettes (`src/presets.js`)

```js
// PALETTES: [{ name: string, colors: string[], colorBlend?: "smooth"|"sharp"|"stepped", colorBalance?: number[] }]
```

## Configurable Options (DEFAULT_OPTIONS in src/main.js)

Core groups: mode, speed, direction, grain, colors, colorBlend, colorBalance, rimLight, meshDetail, foldIntensity, foldScale, foldSpeed, field (octaves, lacunarity, gain, warpStrength, seed, flow, angle, noiseType, fractalType, boxFreq, debug, min, max), pixelRatio, maxFPS, qualityPreset, reactive, mouseRadius, scrollReactive, border, helper.

## Notes

- **Edit `src/` only** — `dist/` is generated by `scripts/release.sh`.
- `spectraGL-noisekit.js` is the unminified source (~1517 lines). It can be edited, but changes must be minified via `release.sh` before publishing.
- There are no tests, linters, or CI configuration.
- Debug modes 0-4 are available via the `field.debug` option for visualization inspection.
